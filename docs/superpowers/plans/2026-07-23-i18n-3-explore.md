**Goal:** Localize Home, Watchlist, Universe, News, Ticker Detail, AI Card,
shared tags, and shared personalization display into `zh-Hant` and English
without translating source/generated content, replaying requests, losing
active workflow state, exposing raw errors, changing Explore behavior or IA,
or rendering the future locale selector.

**Architecture:** Add a statically bundled `explore` namespace and one pure
`explorePresentation` boundary for semantic outcomes, operation-specific
errors, safe diagnostics, and reviewed recovery targets. Each Explore surface
subscribes reactively to its own namespace. `personalizationDisplay.ts`
evolves in place into the sole `common.personalization` presenter, while
Settings, AICard, Research, and Evidence Drawer pass it a namespace-typed
translator. Catch paths retain structured `ApiError` facts rather than
localized text; render-time presenters own copy. No backend, API transport,
CSS, formatter, package, prompt, or data-contract change is authorized.

**Tech Stack:** React 18, TypeScript 5.9, i18next 26.3.6 selector API,
react-i18next 17.0.10, Vitest 4/jsdom, the TypeScript-AST visible-literal
scanner, Vite/Electron, and Playwright/CDP against isolated fake-backed data.

## Locked Implementation Decisions

1. I18N-3 is frontend-only. Backend `src/`, `data_sources/`, backend tests,
   API transport behavior, schemas, prompts, extension code, desktop-main
   code, packages, and lockfiles remain byte-identical to `c0448213`.
2. The public locale selector remains absent. Tests/dev harnesses may use the
   reviewed locale controller or i18n instance; product surfaces render no
   selector, disabled placeholder, or language navigation item.
3. `ui_locale` never enters Card generation, Card translation, Research, or
   another model prompt. Existing explicit Card translation remains a user
   action with the same payload, cache, model, and default.
4. The new `explore` namespace contains exactly `352` non-empty leaves per
   locale. It is statically bundled; no lazy resource loading or Suspense is
   introduced.
5. Common grows from `18` to `32` leaves by adding exactly fourteen
   `common.personalization` leaves: seven stances, four mismatch states, and
   three trace-summary templates.
6. Settings removes exactly eleven duplicate stance/mismatch leaves and falls
   from `713` to `702`. The historical pre-Slice-5 adjusted count node evolves
   from `623` to `612`; its Investor subtree count falls from `151` to `140`.
   Seven Settings-only stance-effect explanations remain in Settings.
7. Research remains exactly `5` leaves. Its two production consumers receive
   only reactive shared-personalization wiring; remaining Research chrome is
   I18N-4 debt and is not pulled into migrated scopes.
8. The resource key set is identical and all leaves are non-empty in both
   locales. Dynamic keys, source values, generated prose, user input, and
   identifiers are forbidden in resources.
9. The final scanner contract is exact `703/656/637/20`, derived by removing
   all `330` owned occurrences and `317` owned signatures, while leaving the
   allowlist unchanged. Migrated scopes are exactly `39`: current `30`, eight
   owned files, and one `src/explore/**` scope.
10. The debt manifest removes owned entries rather than retaining stale
    ceilings. No allowlist entry is added for translatable copy, raw error
    text, tags, source data, or a value merely because both locales spell it
    identically.
11. `personalizationDisplay.ts` remains the single stable module path. Its
    public presenters accept `TFunction<"common">`; they never read the
    i18next singleton. Settings, AICard, Research, and Evidence Drawer pass a
    reactive Common translator.
12. The exact existing node `maps every Investor stance and mismatch state in
    both locales` evolves in place. It proves the shared owner returns all
    seven/four exact terms in both locales, the Settings duplicate paths no
    longer exist, and Settings consumers use the shared owner.
13. `settingsStanceLabel` and `settingsMismatchLabel` are removed, not left as
    delegating wrappers. Shared presentation has one owner, not two functions
    connected by parity tests.
14. `tags.tsx` localizes only known facet/read-only chrome. Tag value, source,
    filter key, editability, CSS class, and unknown stable IDs remain original.
15. `ExploreOperation` is a closed union of exactly 39 IDs listed below.
    Normal error copy is selected from that ID, never from `.message` or an
    English substring.
16. `ApiError` state reads only `status`, `path`, `code`, and `diagnostic`.
    Its legacy `.message` remains untouched for unmigrated consumers and is
    never parsed or rendered by Explore.
17. Safe diagnostic metadata is fail-closed. Status is an integer `100..599`;
    code matches the bounded stable-ID grammar; path is converted to a
    reviewed route template after removing query/fragment and never echoes a
    ticker, search query, run ID, user value, host, or absolute URL.
18. Optional free text is one-line, bounded, and rejected wholesale when it
    resembles credentials, auth material, a traceback/stack, SQL/database
    exception, path, URL/host/IP, HTML/control text, or an over-limit value.
    Unsafe text produces the localized `detail omitted` state. Raw text is not
    a fallback.
19. Recovery destinations are a closed mapping to existing
    `NavigationTarget` Settings sections. Unknown code/status/detail has no
    destination. Missing semantic codes trigger stop-and-amend; English text
    inference is forbidden.
20. `App.tsx` remains the sole Developer Mode and navigation owner. It passes
    capabilities only; no Explore text or second error store moves into App.
    Existing sidecar raw-error state remains the named I18N-5 backlog.
21. Existing silent-degradation paths remain silent. The operation catalog
    can represent list/search/default failures, but I18N-3 does not create a
    new visible banner where current behavior intentionally degrades without
    one.
22. Every visible error and Universe import outcome is stored as semantic
    state. A response completing after a locale switch renders in the current
    locale rather than retaining the locale present at request start.
23. Locale changes preserve component identity, focus, reading context,
    loaded data, filters, pagination, drafts, optimistic state, in-flight work,
    modal/disclosure state, selected Card language, and translated Card result.
    The locale controller PUT is the only request caused by a switch.
24. No component or subtree uses a locale-derived React `key`. Memoized display
    values include `t`/locale dependencies. Reading-position tests require no
    reset, not pixel-identical `scrollTop`.
25. Current loading/empty/degraded/partial/success/error trigger conditions are
    unchanged. Their copy migrates; their state machine does not.
26. Dates, times, numbers, percentages, currencies, compact counts, and
    existing formatter inputs stay unchanged. Formatter policy remains
    explicitly deferred.
27. Existing zh-Hant chrome is byte-preserved except the exact copy ledger
    below. English resources are authored from the terminology authority.
    Stable professional identifiers listed below remain English in both
    locales.
28. No CSS change is pre-authorized. A measured overflow stops implementation
    for a reviewed RED-first deviation with an exact one-owner hunk. Copy may
    not be shortened, clipped, or hidden to make a gate pass.
29. No IA, route, table, filter, pagination, tab, Card, modal, mutation,
    Provider, data, or formatter redesign is authorized.
30. Raw frontend accounting is exactly `+83/-3` (net `+80`), producing
    `90 files / 943 tests`. Focused accounting is `9/85 -> 16/165`.

export type ExploreT = TFunction<"explore">;

  | "home_load_workspace"
