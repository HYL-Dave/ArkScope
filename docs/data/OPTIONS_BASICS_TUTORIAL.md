## 期權合約的標準化

### 「39 交易所, 21 到期日, 113 行使價」是什麼意思？

期權和股票最大的不同：**期權是標準化合約**，就像買賣「規格固定的商品」。

```
股票交易:
├── 你想買 AAPL
└── 只有一種 "AAPL"，在任何交易所都是同一個東西

期權交易:
├── 你想買 AAPL 的 Call Option
└── 但是有「數千種不同的 AAPL Call Option」！
    ├── 到期日不同 → 21 種選擇
    ├── 行使價不同 → 113 種選擇
    └── 交易所不同 → 39 個交易所都有掛牌
```

### 實際例子

```
AAPL 當前股價: $230

你想買「AAPL 看漲期權 (Call)」，但你必須選擇：

1. 選擇到期日 (21 種):
   ├── 2026-01-23 (2天後)     ← 最近
   ├── 2026-01-30 (9天後)
   ├── 2026-02-06 (15天後)
   ├── ...
   └── 2027-01-15 (1年後)     ← 最遠

2. 選擇行使價 (113 種):
   ├── $100  ← 深度價內 (Deep ITM)
   ├── $200  ← 價內 (ITM)
   ├── $230  ← 平價 (ATM) ← 接近當前股價
   ├── $250  ← 價外 (OTM)
   └── $400  ← 深度價外 (Deep OTM)

3. 交易所會自動選擇 (通常用 SMART):
   ├── CBOE (芝加哥期權交易所)
   ├── ISE
   ├── PHLX
   └── ... 共 39 個交易所

所以：
「AAPL 2026-01-23 $230 Call」是一個特定的期權合約
「AAPL 2026-01-23 $235 Call」是另一個完全不同的合約！
```

## 到期日的規則

### 你的問題：「到期日是購買時就固定的？還是從購買日開始計算？」

**答案：到期日是交易所預先設定的，與你的購買時間無關。**

```
錯誤理解 ❌:
├── 我今天買期權
└── 自動到期日 = 今天 + 30天

正確理解 ✅:
├── 交易所預先設定好「固定的到期日列表」
├── 你只能從這些日期中選擇
└── 無論你何時購買，到期日都是那些固定日期
```

### 到期日的規律

美股期權的到期日有標準規則：

```
1. 週期權 (Weekly Options):
   ├── 每週五到期
   ├── 例: 1/23(五), 1/30(五), 2/6(五)...
   └── 最近幾週都有

2. 月期權 (Monthly Options):
   ├── 每月第三個週五到期
   ├── 例: 1月的是 1/16(五), 2月的是 2/20(五)
   └── 這是最「標準」的期權

3. LEAPS (長期期權):
   ├── 到期日在 1-2 年後
   ├── 只有特定月份 (通常是 1月、6月)
   └── 例: 2027-01-15
```

### 實際例子

```
今天是 2026-01-22 (週三)

如果你今天想買 AAPL Call，可選的到期日是：
├── 2026-01-23 (明天週四)  ← 1天後到期，超短期
├── 2026-01-24 (後天週五) ← 2天後到期
├── 2026-01-30 (下週五)   ← 8天後到期
├── 2026-02-20 (2月第三週五) ← 標準月期權
├── ...
└── 2027-01-15            ← LEAPS

如果你下週一 (1/27) 才買同樣的期權：
├── 2026-01-24 已經過期，不能買了
├── 2026-01-30 變成只剩 3 天
└── 其他到期日不變

重點：到期日是「固定時間點」，不是「從購買日算起」
```

## 行使價與實際操作

### 你的問題：「行使時一定要真實買賣股票，還是可以直接拿差價？」

**答案：兩種方式都可以！這叫做「行使」vs「平倉」。**

### 方式一：行使 (Exercise) - 真實買賣股票

```
情境：你持有「AAPL $230 Call」，到期時 AAPL 股價 = $250

行使 Call:
├── 你有權以 $230 買入 100 股 AAPL
├── 你支付: $230 × 100 = $23,000
├── 你獲得: 100 股 AAPL (市值 $25,000)
└── 你的帳戶需要有 $23,000 現金

行使 Put (假設你有「AAPL $230 Put」):
├── 你有權以 $230 賣出 100 股 AAPL
├── 你需要先有 100 股 AAPL
├── 你賣出獲得: $230 × 100 = $23,000
└── 即使市價只有 $200，你還是能賣 $230
```

### 方式二：平倉 (Close Position) - 直接拿差價 ⭐ 最常用

```
情境：你持有「AAPL $230 Call」，現在 AAPL 股價 = $250

平倉 (賣掉期權本身):
├── 你當初買入期權花了: $5.00 × 100 = $500
├── 現在期權價值約: $20.00 × 100 = $2,000
├── 你直接賣掉期權
└── 獲利: $2,000 - $500 = $1,500

重點：
├── 90%+ 的期權交易者都是「平倉」而不是「行使」
├── 平倉不需要準備大量資金買股票
├── 平倉可以隨時操作，不必等到到期日
└── 行使通常只在到期日才做，而且需要足夠資金
```

### 實際操作流程

```
大多數人的操作方式：

1. 買入期權
   └── 花 $500 買入「AAPL $230 Call」

2. 等待 (持有期間)
   ├── 如果 AAPL 漲 → 期權價值上升
   └── 如果 AAPL 跌 → 期權價值下降

3. 平倉 (隨時可以)
   ├── 股價漲到 $250 → 期權值 $2,000 → 賣掉獲利 $1,500
   ├── 股價跌到 $220 → 期權值 $200 → 賣掉止損 -$300
   └── 不需要真的買賣 AAPL 股票！

4. 如果不平倉，到期時:
   ├── 價內 (ITM): 自動行使，需要資金
   ├── 價外 (OTM): 自動作廢，損失全部權利金
   └── 所以大多數人會提前平倉
```

### 為什麼大多數人選擇平倉？

```
假設：AAPL $230 Call，股價漲到 $250

行使：
├── 需要準備: $23,000 買股票
├── 然後賣股票獲得: $25,000
└── 淨獲利: $2,000 - 權利金

平倉：
├── 只需要: $0 (不需要額外資金)
├── 直接賣掉期權: 獲得 ~$2,000
└── 淨獲利: ~$2,000 - 權利金

結論：效果幾乎一樣，但平倉更簡單、不需要大量資金
```

## 報價延遲的原因

### 你的問題：「為什麼期權有 15 分鐘延遲？股票沒有這麼長的延遲？」

**答案：這是數據訂閱的商業模式，不是技術限制。**

### 數據訂閱層級

```
股票數據 (以 IBKR 為例):
├── 免費: 15-20 分鐘延遲報價
├── Level 1 ($1.50/月): 即時 NBBO (最佳買賣價)
└── Level 2 ($16-25/月): 即時深度 (所有掛單)

期權數據 (OPRA):
├── 免費: 無數據
├── OPRA ($1.50/月): 15 分鐘延遲報價 ← 你現在訂的
└── OPRA + 股票數據: 即時報價
```

### 為什麼期權更嚴格？

```
原因 1: 數據量巨大
├── AAPL 股票: 1 個報價
├── AAPL 期權: 21 到期日 × 113 行使價 × 2 (Call/Put) = 4,746 個報價！
└── 整個市場的期權報價量是股票的數百倍

原因 2: 商業模式
├── 交易所靠賣數據賺錢
├── 專業交易者需要即時數據
├── 散戶用延遲數據「試用」
└── 想要即時數據就要付費

原因 3: 你訂閱的 OPRA 層級
├── OPRA $1.50/月 是「基礎版」
├── 它提供延遲報價 (15分鐘)
├── 要即時報價，還需要訂閱「underlying 股票數據」
└── 因為期權價格計算需要股票即時價格
```

### 對你的影響

```
15 分鐘延遲的影響:

研究分析: ✅ 完全夠用
├── 看期權鏈結構
├── 分析歷史數據
└── 學習期權定價

中長期交易: ✅ 夠用
├── 持有數天或數週的期權
├── 15 分鐘延遲影響很小
└── 用限價單可以避免買貴

短期交易 (日內): ⚠️ 可能不夠
├── 期權價格變化快
├── 15 分鐘前的報價可能差很多
└── 可能錯過最佳進出點

做市/高頻: ❌ 完全不夠
├── 需要毫秒級數據
├── 15 分鐘延遲 = 無法操作
└── 需要專業級訂閱
```

## Greeks 詳解

**Greeks** 是衡量期權風險和敏感度的指標，用希臘字母命名。

### 數學定義總覽

| Greek | 數學定義 | 直白解釋 |
|-------|---------|---------|
| **Delta (Δ)** | ∂V/∂S | 期權價格對股價的一階導數 |
| **Gamma (Γ)** | ∂²V/∂S² | 期權價格對股價的二階導數 (Delta 的變化率) |
| **Theta (Θ)** | ∂V/∂t | 期權價格對時間的導數 (通常為負) |
| **Vega (ν)** | ∂V/∂σ | 期權價格對波動率的導數 |
| **Rho (ρ)** | ∂V/∂r | 期權價格對利率的導數 (通常忽略) |

其中：V = 期權價格，S = 股價，t = 時間，σ = 波動率，r = 利率

### 數值範圍

| Greek | Call | Put | 說明 |
|-------|------|-----|------|
| **Delta** | 0 到 +1 | -1 到 0 | Call 正相關，Put 負相關 |
| **Gamma** | 永遠 ≥ 0 | 永遠 ≥ 0 | 兩者都是正的 |
| **Theta** | 通常 < 0 | 通常 < 0 | 時間流逝損失價值 |
| **Vega** | 永遠 ≥ 0 | 永遠 ≥ 0 | 波動率增加對買方有利 |

### Delta (Δ) - 方向敏感度

```
定義: 股價變動 $1 時，期權價格變動多少

範例:
├── Delta = 0.50 的 Call
├── 股價從 $230 漲到 $231 (+$1)
└── 期權價格約漲 $0.50

Delta 的範圍:
├── Call: 0 到 +1.0
│   ├── ATM Call 約 0.50
│   ├── ITM Call 接近 1.0
│   └── OTM Call 接近 0
│
└── Put: -1.0 到 0
    ├── ATM Put 約 -0.50
    ├── ITM Put 接近 -1.0
    └── OTM Put 接近 0

實用意義:
├── Delta 0.50 = 你「擁有」50 股的效果
├── Delta 0.80 = 你「擁有」80 股的效果
└── 可以用來計算避險比例
```

### Gamma (Γ) - Delta 的變化率

```
定義: 股價變動 $1 時，Delta 變動多少

重要: Gamma 永遠是正數，無論 Call 或 Put！

範例 (Call):
├── Delta = 0.50, Gamma = 0.05
├── 股價從 $230 漲到 $231
├── 新 Delta = 0.50 + 0.05 = 0.55
└── 期權對股價上漲的反應加速了

範例 (Put):
├── Delta = -0.50, Gamma = 0.05
├── 股價從 $230 跌到 $229
├── 新 Delta = -0.50 - 0.05 = -0.55
└── 期權對股價下跌的反應加速了

Gamma 與 Call/Put 的關係:
├── Call: 股價漲 → Delta 變大 → 期權漲更快
├── Call: 股價跌 → Delta 變小 → 期權跌變慢
├── Put:  股價跌 → Delta 變更負 → 期權漲更快
└── Put:  股價漲 → Delta 變接近 0 → 期權跌變慢

總結: Gamma 讓期權對「有利方向」的反應加速

Gamma 的特性:
├── ATM 期權的 Gamma 最大 (最敏感)
├── 越接近到期，ATM 的 Gamma 越大
└── OTM 和 ITM 的 Gamma 較小

實用意義:
├── 高 Gamma = 期權價格變化「加速」
└── 快到期的 ATM 期權波動最劇烈
```

### Theta (Θ) - 時間衰減

```
定義: 每過一天，期權損失多少價值

範例:
├── 期權價格 = $5.00
├── Theta = -0.10
├── 明天 (其他條件不變)
└── 期權價格 ≈ $4.90

Theta 的特性:
├── Theta 幾乎總是負數 (期權買方的敵人)
├── 越接近到期，Theta 越大 (衰減越快)
├── ATM 期權的 Theta 最大
└── LEAPS 的 Theta 很小

實用意義:
├── 買期權 → Theta 對你不利 (每天虧錢)
├── 賣期權 → Theta 對你有利 (每天賺錢)
└── 這就是為什麼長期持有期權風險高
```

### Vega (ν) - 波動率敏感度

```
定義: 隱含波動率 (IV) 變動 1% 時，期權價格變動多少

範例:
├── 期權價格 = $5.00
├── Vega = 0.15
├── IV 從 30% 上升到 31%
└── 期權價格 ≈ $5.15

Vega 的特性:
├── ATM 期權的 Vega 最大
├── 到期時間越長，Vega 越大
└── 波動率上升 → 所有期權都變貴

實用意義:
├── 買期權前看 IV 是否太高
├── IV 太高 = 期權太貴
└── 財報前 IV 通常飆高，財報後 IV 崩跌 (IV Crush)
```

### IV (Implied Volatility) - 隱含波動率

```
定義: 市場「預期」的未來股價波動程度

範例:
├── AAPL IV = 25%
└── 市場預期 AAPL 年化波動約 25%

├── TSLA IV = 60%
└── 市場預期 TSLA 波動更劇烈

IV 的用途:
├── 判斷期權是否「貴」或「便宜」
├── IV 高 → 期權貴 (市場預期大波動)
├── IV 低 → 期權便宜 (市場預期平靜)
└── 可以和歷史波動率 (HV) 比較

IV 的變化:
├── 財報前: IV 飆升 (不確定性高)
├── 財報後: IV 崩跌 (不確定性消除)
└── 黑天鵝事件: IV 暴漲
```

### Greeks 綜合範例

```
你持有: AAPL $230 Call
├── Delta: 0.55  → 股價漲 $1，期權漲 $0.55
├── Gamma: 0.03  → Delta 會隨股價變化
├── Theta: -0.08 → 每天損失 $0.08 (時間衰減)
├── Vega: 0.12   → IV 漲 1%，期權漲 $0.12
└── IV: 42%      → 市場預期中等波動

情境分析:
├── 股價漲 $5 → 期權約漲 $2.75 (Delta 效果)
├── 持有 10 天 → 損失約 $0.80 (Theta 效果)
├── IV 從 42% 降到 35% → 損失約 $0.84 (Vega 效果)
└── 最終損益 = $2.75 - $0.80 - $0.84 = $1.11
```

## 期權研究分析

### 「期權研究分析」具體是什麼？

```
期權研究分析 = 使用數據做出交易決策

具體包括:

1. 分析期權鏈
   ├── 哪些行使價/到期日的成交量大？
   ├── Put/Call 比例是多少？
   └── 市場偏看漲還是看跌？

2. 分析定價
   ├── 某個期權是否被低估/高估？
   ├── IV 相對歷史是高還是低？
   └── 是否有套利機會？

3. 分析 Greeks
   ├── 我的部位對股價變動有多敏感？
   ├── 時間衰減會損失多少？
   └── 如何調整避險？

4. 分析 Options Flow
   ├── 大戶在買什麼？
   ├── 有沒有異常成交？
   └── Smart Money 的方向？
```

### 你用 OPRA 可以做的研究

```
✅ 可以做:
├── 查看完整期權鏈 (所有到期日、行使價)
├── 查看延遲報價 (Bid, Ask, Last)
├── 查看 Greeks (Delta, Gamma, Theta, Vega, IV)
├── 分析期權定價是否合理
├── 學習和回測策略
└── 中長期交易決策

❌ 無法做 (需要其他數據):
├── 即時搶進搶出
├── Sweep/Block 偵測 (需要逐筆數據)
└── 日內短線交易
```

## Options Flow 與 Sweep/Block

### 什麼是 Options Flow？

```
Options Flow = 追蹤「大戶」的期權交易行為

原理:
├── 大戶 (機構、對沖基金) 有資訊優勢
├── 他們的交易行為可能預示未來走勢
└── 透過監測他們的交易，散戶可以「跟單」
```

### 什麼是 Sweep？

```
Sweep (掃單) = 大戶急著買入，一次掃光多個交易所的賣單

正常交易:
├── 我要買 1,000 張 AAPL Call
├── 在 CBOE 下單，等待成交
└── 可能要等幾分鐘才買到

Sweep 交易:
├── 我要買 1,000 張 AAPL Call，而且很急！
├── CBOE 只有 300 張 → 買光
├── ISE 只有 400 張 → 買光
├── PHLX 剩下的 → 買光
└── 5 秒內跨 3 個交易所買完

為什麼 Sweep 重要:
├── 表示買家非常急迫
├── 急迫 = 可能有內幕消息或強烈預期
└── Sweep 是最強的看漲/看跌信號之一
```

### 什麼是 Block？

```
Block Trade (大宗交易) = 單筆超大金額的交易

定義:
├── 單筆 > $1,000,000
├── 或單筆 > 100 張合約
└── 通常是機構的交易

Block 的意義:
├── 小額 = 散戶，參考價值低
├── 大額 = 機構，可能知道什麼
└── 特別是沒有對應新聞時的 Block
```

### 為什麼 IBKR 無法偵測 Sweep/Block？

```
偵測 Sweep 需要:
├── 每一筆成交的「時間」(精確到秒)
├── 每一筆成交的「交易所」
└── 即時的逐筆數據流

IBKR 的限制:
├── reqTickByTickData 對期權返回 Error 10189
├── 說明: "逐筆交易數據請求不適用於此期權"
└── IBKR 不提供期權的逐筆成交數據

結論:
├── 無法自己偵測 Sweep (不知道每筆成交的交易所)
├── 只能看「聚合」後的報價，看不到「過程」
└── 所以需要第三方服務如 Unusual Whales
```

### Unusual Whales 提供什麼？

```
Unusual Whales ($50/月):
├── 已處理好的 Sweep 信號
├── 已處理好的 Block 信號
├── 標示看漲/看跌方向
├── 歷史數據可回測
├── 國會議員交易追蹤 (獨家)
└── API 可整合到你的系統

你用它可以:
├── 收到「AAPL 出現 $2M Call Sweep」警報
├── 知道「機構在買入」
└── 決定是否跟進
```

├── 額外訂閱: 可選
