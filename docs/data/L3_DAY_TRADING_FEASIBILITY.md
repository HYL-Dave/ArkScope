## 重要發現：L3 細分類別

### L3 不是單一類別！

傳統將「日內交易」視為單一類別是不精確的。對於 AI Agent，L3 應細分為：

```
L3 日內交易細分:

L3a (快速日內)          L3b (標準日內)          L4 (短波段)
1-5 分鐘區間            15-30 分鐘區間          1-4 小時區間
     │                       │                      │
     │                       │                      │
     ▼                       ▼                      ▼
需要即時數據             即時數據也建議！         即時數據也建議！
LLM 時間緊張             LLM 時間充裕 ⭐          LLM 時間非常充裕
與 HFT 競爭              避開 HFT                避開 HFT
```

### L3a vs L3b 關鍵差異 (更正版)

| 特性 | L3a (1-5分鐘) | L3b (15-30分鐘) | 影響 |
|------|--------------|----------------|------|
| **數據延遲容忍** | < 1 分鐘 | 15 分鐘可接受 | 兩者都建議即時 |
| **LLM 分析時間** | 緊張 (1-2分鐘) | **充裕 (5-10分鐘)** | ✅ **L3b 核心優勢** |
| **每日決策次數** | 50-100+ | ~26 次 (15分鐘) | ✅ L3b 成本可控 |
| **與 HFT 競爭** | 直接競爭 | **避開首波** | ✅ **L3b 核心優勢** |
| **需要即時數據？** | ✅ 必須 | 建議有 (~$10) | 成本差異小 |
| **需要 WebSocket？** | ✅ 必須 | 可選 (建議有) | L3b 更靈活 |
| **數據月費** | ~$90 | ~$90 | **差異很小** |

### 🎯 15 分鐘區間 = AI Agent 甜蜜點

**15 分鐘交易區間 (L3b) 是 AI Agent 的最佳選擇：**

1. **保留日內交易機會** - 每日 26 個決策點
2. **LLM 分析時間充裕** - 每循環 5-10 分鐘深度分析 ⭐ 核心優勢
3. **避開 HFT 競爭** - 不追求首波反應 ⭐ 核心優勢
4. **決策品質更高** - 有時間整合多因子
5. **可預抓 ~80% 數據** - 批次處理友善

**注意**: L3b 的優勢**不是省數據費**（差異只有 ~$10/月），而是**分析品質**。

# 低延遲處理流程
async def signal_pipeline(signal):
    # Step 1: 規則篩選 (< 10ms)
    if not passes_basic_rules(signal):
        return

    # Step 2: 快速特徵提取 (< 50ms)
    features = extract_features(signal)

    # Step 3: 小模型快速評估 (< 2s)
    quick_assessment = await small_model.evaluate(features)

    if quick_assessment.confidence > 0.8:
        # Step 4a: 立即執行小倉位
        execute_small_position(signal, quick_assessment)

        # Step 4b: 背景深度分析
        asyncio.create_task(deep_analysis(signal))
    else:
        # 僅記錄，不執行
        log_signal(signal, quick_assessment)
```

        max_stocks=20
    )

    # 3. 生成每支股票的交易規則
    for stock in watchlist:
        rules = await llm.generate_trading_rules(stock)
        save_rules_to_engine(stock, rules)

    return watchlist
```

## 風險與限制

### L3 特有風險

| 風險 | 說明 | 緩解措施 |
|------|------|---------|
| **過度交易** | 更多機會 = 更多交易 = 更多手續費 | 設定每日交易次數上限 |
| **情緒干擾** | 盤中波動影響判斷 | 嚴格遵循盤前規則 |
| **技術故障** | 斷線、API 錯誤 | 自動熔斷 + 手動備援 |
| **滑點** | 快速市場中執行價差 | 限價單 + 流動性篩選 |
| **PDT 規則** | 美股帳戶 < $25,000 限制日內交易 | 確保資金充足或使用現金帳戶 |

### AI Agent 在 L3 的限制

```
無法做到:
├── 毫秒級反應 (LLM 物理限制)
├── 與 HFT 競爭首波反應
├── 持續 6.5 小時高頻決策 (成本過高)
└── 預測完全隨機的價格波動

可以做到:
├── 盤前深度準備 (優勢所在)
├── 識別有意義的訊號 (vs 噪音)
├── 5-30 分鐘窗口的機會
└── 持續學習和優化規則
```

### 需要的程式碼調整

```python
class TradingSessionManager:
    """24 小時交易時段管理"""

    def get_session_tier(self, time: datetime) -> int:
        """根據時間返回交易層級 (1=高, 2=中, 3=低)"""
        hour = time.hour  # ET timezone

        if 9 <= hour < 16:  # 9:30am - 4:00pm
            return 1
        elif 16 <= hour < 20:  # 4:00pm - 8:00pm
            return 2
        else:  # 8:00pm - 9:30am
            return 3

    def get_position_multiplier(self, tier: int) -> float:
        """根據層級調整倉位大小"""
        return {1: 1.0, 2: 0.5, 3: 0.25}[tier]

    def get_spread_threshold(self, tier: int) -> float:
        """根據層級調整 spread 門檻"""
        return {1: 0.002, 2: 0.001, 3: 0.0005}[tier]  # 0.2%, 0.1%, 0.05%
```
