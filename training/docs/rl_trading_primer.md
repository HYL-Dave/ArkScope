# RL 交易入門 — 從零理解本專案在做什麼

寫給不熟悉 RL 的自己，用交易的語言解釋。

## 1. 傳統量化 vs RL 量化

### 傳統量化

你自己寫規則：

```
如果 RSI < 30 且情緒分數 > 4 → 買入
如果 RSI > 70 或情緒分數 < 2 → 賣出
```

問題：規則是你想的，市場不一定買單。而且參數怎麼調？閾值 30 和 70 是最好的嗎？

### RL 量化

你不寫規則，讓 agent 自己學：

```
你：這是 134 支股票的價格、技術指標、LLM 情緒分數（共 1475 個數字）
     你可以對每支股票決定買賣量（134 個數字）
     每天結束我告訴你賺了還是虧了

Agent：（反覆練習幾百萬步）我學到了：
       - 當某種 state 組合出現時買入比較好
       - 當另一種組合出現時不動比較好
       - 但我不會告訴你學到了什麼規則（黑箱）
```

## 2. 核心組件

### Environment（環境）= 模擬的股市

`stocktrading_llm.py` 就是一個模擬器：

```
初始狀態：2022-01-03，100 萬現金，0 持股

每天：
  1. agent 看到 state（1475 維向量）
  2. agent 輸出 action（134 維，每支股的買賣量）
  3. 環境執行交易（扣手續費）
  4. 環境給出 reward（portfolio 變化）
  5. 推進到下一個交易日

終止：走完所有交易日
```

### Agent（代理人）= 學習中的交易員

兩個神經網路組成：

```
Policy 網路（Actor）：
  輸入 1475 維 state → 輸出 134 維 action
  「在這種市況下，我應該怎麼交易」

Value 網路（Critic）：
  輸入 1475 維 state → 輸出 1 個數字
  「在這種市況下，我預計從現在到結束還能賺多少」
```

### PPO 演算法 = 學習方法

PPO 是 agent 更新策略的方法。類比：

```
新手交易員的學習過程：

1. 用現有策略交易一段時間（收集經驗，20000 步）
2. 回顧：哪些交易賺了？哪些虧了？
3. 用 Advantage（優勢）量化：
   - 某次買入的結果比「預期」好 → Advantage > 0 → 強化這個行為
   - 某次賣出的結果比「預期」差 → Advantage < 0 → 抑制這個行為
4. 但每次調整幅度不能太大（PPO Clip）→ 穩定學習
5. 回到步驟 1，用更新後的策略繼續交易
```

## 3. 情緒分數在哪裡起作用

情緒分數（llm_sentiment）影響交易的兩個層面：

### 層面 A：State 的一部分

agent 看到的 1475 維 state 裡包含 134 個情緒分數。
神經網路可以學到「情緒高 + 價格低 = 買入信號」這類模式。

### 層面 B：Action 的調節器（Sentiment Scaling）

環境在執行交易前，會根據情緒調整 action 幅度：

```
agent 說：買入 NVDA 50 股

環境檢查情緒分數：
  情緒 = 5（非常正面）且 agent 要買入 → 匹配！
  → 實際買入量 = 50 × 1.10 = 55 股（強化匹配的行動）

  情緒 = 1（非常負面）且 agent 要買入 → 不匹配！
  → 實際買入量 = 50 × 0.90 = 45 股（抑制矛盾的行動）
```

這個機制讓情緒分數成為「額外的安全網」：
- agent 的策略不好也會被情緒信號部分修正
- agent 的策略好，情緒信號幫它加碼

## 4. 訓練的完整流程

```
┌─────────────────────────────────┐
│ Step 0: 準備資料                  │
│                                 │
│ 新聞 → LLM 評分 → sentiment/risk │
│ + yfinance OHLCV + 技術指標      │
│ → 合併為 train.csv / trade.csv   │
└────────────────┬────────────────┘
                 ↓
┌─────────────────────────────────┐
│ Step 1: 訓練 (train.csv)         │
│                                 │
│ Agent 在 2022-2024 的模擬環境裡   │
│ 反覆練習（100 epochs × 20k 步）  │
│ 每次根據 reward 調整策略           │
│ → 輸出 model.pth（學好的策略）    │
└────────────────┬────────────────┘
                 ↓
┌─────────────────────────────────┐
│ Step 2: 回測 (trade.csv)         │
│                                 │
│ 把訓練好的策略放到 2025-2026 的    │
│ 「從未見過」的行情中執行           │
│ → Sharpe, Max Drawdown 等指標    │
│                                 │
│ 這一步才是真正的成績單            │
│ （Train 賺不代表 Trade 也能賺）   │
└────────────────┬────────────────┘
                 ↓
┌─────────────────────────────────┐
│ Step 3: 判斷                     │
│                                 │
│ Trade Sharpe > 0.5 且沒過擬合？   │
│ → 考慮 Phase 2 全資料重訓部署     │
│                                 │
│ Trade 表現差？                    │
│ → 調參、換資料分割、檢查環境設定  │
└─────────────────────────────────┘
```

## 5. PPO vs CPPO

| | PPO | CPPO |
|---|---|---|
| 目標 | 最大化回報 | 最大化回報 **且** 控制風險 |
| 用什麼分數 | llm_sentiment | llm_sentiment + llm_risk |
| 怎麼控制風險 | 不管（只要賺就好） | CVaR 約束：限制尾部虧損 |
| 適合場景 | 追求收益 | 需要控制最大回撤的場景 |

CPPO 的 CVaR 約束用白話說：

```
PPO：找到能賺最多的策略
CPPO：找到能賺最多的策略，但最差的 15% 情況不能虧太多
```

## 6. 為什麼不直接用情緒分數寫規則

你可能會問：既然有情緒分數，為什麼不直接 `if sentiment > 4: buy`？

因為 RL agent 可以學到更複雜的模式：

```
規則策略只能：
  if sentiment > 4 and RSI < 30: buy

RL agent 可以學到：
  「當 NVDA 的情緒 = 4, RSI = 35, MACD 在上升,
   同時 AMD 的情緒 = 2 且整體市場波動率高時，
   買入 NVDA 30 股並同時減持 AMD 10 股」
```

agent 考慮的是**所有 1475 個維度的組合**，而不是你能想到的幾個規則。
代價是黑箱 — 你不知道它學到了什麼具體規則。

## 7. 可能遇到的問題

### 過擬合 (Overfitting)

Agent 在 train 期表現很好（記住了歷史行情），但在 trade 期表現差。

識別：Train Sharpe > 2 但 Trade Sharpe < 0 → 幾乎確定過擬合。

應對：
- 更多 train 資料（覆蓋更多市場環境）
- 減小網路（`--hid 256`）
- 減少 epochs（可能 50 就夠了，100 反而過擬合）
- 增大 entropy 保持探索

### Policy Collapse

Agent 學到一個「安全但無用」的策略，比如永遠不交易（虧不了也賺不了）。

識別：EpRet ≈ 0, total_trades 很低, Entropy 接近 0。

應對：
- 確認 reward 設計合理
- 檢查 sentiment scaling 沒有壓制所有行動

### 訓練不穩定

EpRet 大幅震盪，無法收斂。

識別：AverageEpRet 在正負之間反覆跳動。

應對：
- 降低學習率（`--lr 1e-5`）
- 降低 clip_ratio
- 增加 steps_per_epoch（每次收集更多經驗）
