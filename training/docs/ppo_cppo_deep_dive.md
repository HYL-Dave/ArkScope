# PPO 與 CPPO 深入解析

本文件從最底層開始，逐步解釋 PPO 和 CPPO 的原理。
理解這些原理是調參的基礎 — 不知道機制就不知道該調什麼。

**前置知識**：假設你理解 MDP 的四個組件（State, Action, Reward, Transition）。
如果不熟悉，先看 `rl_trading_primer.md` 的第二層。

---

## 第一步：Policy Gradient — PPO 的祖先

### 策略是一個機率分佈

Policy 是一個神經網路，輸入 state，輸出 action 的**機率分佈**：

```
state (1475維) → 神經網路 → μ (134維的均值)
                           → σ (134維的標準差，可學習)

實際 action 從 Normal(μ, σ) 抽樣
```

為什麼用機率分佈而不是直接輸出確定的 action？因為**需要探索**。
初期 agent 不知道什麼是好的，需要隨機嘗試各種交易，才能發現哪些有效。

```
σ 大 → 每天的交易決策隨機性高 → 探索（嘗試各種組合）
σ 小 → 每天的交易決策很固定 → 利用（執行已知最好的策略）
```

訓練過程中，σ 應該自然從大（探索）逐漸縮小到一個穩定值（利用為主，保留少量探索）。
對應訓練 log 裡的 **Entropy** 指標 — Entropy 下降 = σ 在縮小。

### 最樸素的更新方式

REINFORCE 演算法（1992，Policy Gradient 的起點）：

```
1. 用現有策略跑一整個 episode（752 天）
2. 記錄每一步的 (state, action, reward)
3. 計算每個 action 的「總回報」R
4. 更新策略：
   - R > 0 的 action → 提高它的機率（梯度上升）
   - R < 0 的 action → 降低它的機率（梯度下降）
```

數學上：

```
梯度 = E[R × ∇log π(a|s)]

R = 回報（正/負）
∇log π(a|s) = action 機率的梯度方向
乘起來 = 回報加權的梯度 → 好的 action 被推高，壞的被壓低
```

### 問題：方差太大（功勞歸屬不清）

假設某天 agent 買了 NVDA 賺了錢。但「賺了」的原因可能是：
- agent 的決策確實好（應該歸功）
- 整個市場都在漲（跟 agent 的決策無關）

如果把市場上漲的功勞歸給 agent，它會錯誤地強化這個行為。
**這就是高方差問題** — 學習信號太嘈雜，策略會震盪不收斂。

解法 → 引入 Advantage。

## 第二步：Advantage — 解決功勞歸屬

### Value 函數 V(s)

引入第二個神經網路 — **Critic**（評論家）：

```
state (1475維) → Value 網路 → V(s)：一個數字

含義：「從這個 state 開始，用現有策略一直走到結束，預計能賺多少」
```

V(s) 就像一個**基準線** — 「在這個市場狀態下，正常水平大概能賺這麼多」。
這個基準線扣除了市場本身的影響，只留下 agent 決策的貢獻。

### Advantage = 實際 - 預期

```
A(s, a) = 實際拿到的回報 - V(s) 預測的回報

A > 0：這個 action 比「正常水平」好 → 強化
A < 0：這個 action 比「正常水平」差 → 抑制
A ≈ 0：跟預期差不多 → 不怎麼調整
```

具體例子：

```
情況 A：市場漲 5%
  V(s) 預測：+5%（基於市場狀態）
  agent 買 NVDA 實際賺了 +8%
  Advantage = 8% - 5% = +3% → 真正的貢獻是 3%

情況 B：市場跌 3%
  V(s) 預測：-3%
  agent 買 NVDA 只虧了 -1%
  Advantage = -1% - (-3%) = +2% → 在逆境中做得比預期好！

情況 C：市場漲 5%
  V(s) 預測：+5%
  agent 做了一堆交易，結果只賺了 +2%
  Advantage = 2% - 5% = -3% → 瞎折騰，不如不動
```

這就是 **Actor-Critic** 架構的核心：
- **Actor**（Policy 網路）：決定行動
- **Critic**（Value 網路）：評估行動是比預期好還是差
- 用 Advantage（而非原始回報）更新 Actor → 信號更乾淨

### GAE-Lambda：精細的 Advantage 估計

> **調參相關**：`lam` (λ) 參數控制偏差-方差 tradeoff。

單步 Advantage 太嘈雜（方差高），純用 V(s) 估計又有偏差。
GAE 是一種加權平均，平衡兩者：

```
δ_t = r_t + γ × V(s_{t+1}) - V(s_t)     ← 單步 TD 誤差

A_t = δ_t + (γλ)δ_{t+1} + (γλ)²δ_{t+2} + ...

γ = 0.995（折扣因子）
λ = 0.95（我們的設定）
```

λ 的效果：

```
λ → 1.0：更多依賴「實際觀察到的回報」（高方差，低偏差）
           適合：reward 信號清晰、episode 短的環境
λ → 0.0：更多依賴「V 網路的預測」（低方差，高偏差）
           適合：V 網路已經很準的情況
λ = 0.95：偏重實際觀察，但用 V 預測做適度修正（常見默認值）
```

**什麼時候需要調 λ**：如果訓練震盪嚴重（StdEpRet 很大），可以降低 λ（例如 0.90）
讓 Advantage 更依賴 V 的預測，犧牲一些精度換穩定性。

## 第三步：PPO — 安全的策略更新

### 問題：更新太大會崩壞

普通 Policy Gradient 的更新幅度沒有限制。如果某個 action 的 Advantage 很大，
策略可能一步跳到極端位置，下一輪就崩了（**policy collapse**）。

類比：

```
交易員學到「昨天重倉 NVDA 賺了 30%」
→ 不加限制的學習：「以後永遠全倉 NVDA！」
→ 下次 NVDA 跌了 → 爆倉 → 策略崩壞

PPO 的做法：「NVDA 確實好，多買一點，但不要太激進」
```

### PPO 的 Clip 機制

PPO 引入 **ratio**（新舊策略的機率比）：

```
ratio = π_new(a|s) / π_old(a|s)

ratio = 1.0 → 新舊策略一樣（沒更新）
ratio = 1.5 → 新策略讓這個 action 的機率提高了 50%
ratio = 0.5 → 新策略讓這個 action 的機率降低了 50%
```

PPO 的損失函數：

```
L = min(ratio × A,  clip(ratio, 1-ε, 1+ε) × A)

我們的 ε = 0.7，clip 範圍 = [0.3, 1.7]
（標準 PPO 用 ε = 0.2，範圍 [0.8, 1.2]。我們更激進。）
```

**min 的作用**（分兩個場景看）：

**場景 1：A > 0（好的 action，想提高機率）**

```
ratio 想往上推（提高好 action 的機率）
但被 clip 在 1.7 封頂
→ 即使這個 action 再好，機率最多提高到 1.7 倍
→ 防止過度強化一個行為

gradient 只在 ratio ∈ [1.0, 1.7] 時有效
超過 1.7 → gradient = 0（不再推了）
```

**場景 2：A < 0（壞的 action，想降低機率）**

```
ratio 想往下壓（降低壞 action 的機率）
但被 clip 在 0.3 封底
→ 即使這個 action 再差，機率最多壓到 0.3 倍
→ 防止完全消滅某個行為（保留探索空間）

gradient 只在 ratio ∈ [0.3, 1.0] 時有效
低於 0.3 → gradient = 0（不再壓了）
```

圖示：

```
     L（損失）

     ╱‾‾‾‾‾‾‾‾               ‾‾‾‾‾‾‾‾╲
     ╱   A > 0                  A < 0   ╲
────╱────────────────────────────────────╲────
   0.3        1.0        1.7
              ratio →

ratio 在 [0.3, 1.7] 之間 → 正常梯度
ratio 超出範圍 → 梯度為 0（停止推動）
```

> **調參相關**：`clip_ratio` (ε) 控制每次更新的最大幅度。
> - ε 太小（0.1）→ 學得慢，但穩定
> - ε 太大（0.9）→ 學得快，但容易崩
> - 我們用 0.7（偏激進），如果訓練不穩可以降到 0.3-0.5

### 第二道保險：KL Early Stopping

Clip 限制的是**單個 action** 的更新幅度。但如果很多 action 同時被推到 clip 邊界，
整體策略變化可能還是很大。所以加一道**全局**檢查：

```
每做一步 gradient update：
  計算新舊策略的 KL 散度（衡量兩個分佈的整體差異）
  如果 KL > 1.5 × target_kl
  → 立刻停止本輪所有更新

我們的 target_kl = 0.35
觸發閾值 = 1.5 × 0.35 = 0.525
```

對應訓練 log：

```
StopIter = 10  → 第 10 步就停了（策略變化太大，提前剎車）
StopIter = 99  → 跑滿 100 步（策略變化溫和，充分更新）
```

> **調參相關**：`target_kl` 控制全局更新幅度。
> - 太小（0.01）→ 幾乎每輪都 early stop，學不動
> - 太大（1.0）→ 形同虛設，失去保護
> - 我們用 0.35（偏大），標準 PPO 用 0.01-0.05

### 一個完整的 PPO Epoch

```
┌─── 1. 收集經驗 ──────────────────────────────────┐
│                                                   │
│  8 個 MPI worker 各用 π_old 跑 2,500 步            │
│  共收集 20,000 步的 (s, a, r, V(s), log π(a|s))   │
│                                                   │
└───────────────────────┬───────────────────────────┘
                        ↓
┌─── 2. 計算 Advantage ────────────────────────────┐
│                                                   │
│  用 GAE-Lambda 計算每步的 A_t                      │
│  正規化：A = (A - mean) / std                     │
│  → 8 個 worker 的 mean/std 用 MPI 平均             │
│                                                   │
└───────────────────────┬───────────────────────────┘
                        ↓
┌─── 3. 更新 Policy（最多 100 步）─────────────────┐
│                                                   │
│  for i in range(100):                             │
│    ratio = π_new(a|s) / π_old(a|s)                │
│    L = min(ratio × A, clip(ratio) × A)            │
│    L.backward()                                   │
│    MPI 平均所有 worker 的 gradient                  │
│    optimizer.step()                                │
│    if KL > 0.525: break   ← early stopping        │
│                                                   │
└───────────────────────┬───────────────────────────┘
                        ↓
┌─── 4. 更新 Value（100 步）───────────────────────┐
│                                                   │
│  for i in range(100):                             │
│    loss = (V(s) - 實際回報)²                       │
│    loss.backward()                                │
│    MPI 平均 gradient                               │
│    optimizer.step()                                │
│                                                   │
└──────────────────────────────────────────────────┘

100 個 epoch = 重複以上步驟 100 次
每次都用更新後的策略收集新經驗
```

## 第四步：CPPO — 在 PPO 上加風險約束

### PPO 的問題：只看期望，不管風險

PPO 最大化的是**期望回報**（平均值）。這意味著：

```
策略 A：平均賺 20%，但有 10% 機率虧 50%
策略 B：平均賺 15%，最差只虧 10%

PPO 選 A（平均回報高）
但作為投資者，你可能想要 B（最差情況可控）
```

在金融裡這個問題尤其嚴重 — 一次 -50% 的 drawdown 就可以讓你出局，
不管之前賺了多少。

### VaR 和 CVaR — 量化尾部風險

```
把策略跑 100 次的日回報從小到大排列：

-8%, -6%, -5%, -4%, -3%, ..., +1%, +2%, +3%, +4%, +5%
 ↑_________________↑
 最差的 5%（5 次）

VaR (95%)  = -5%     ← 第 5 個最差的值（門檻）
CVaR (95%) = -5.8%   ← 最差 5 次的「平均」（門檻以下的平均）
```

CVaR 比 VaR 更嚴格 — VaR 只看門檻值，CVaR 看門檻以下的平均。
CVaR 捕捉了「最差情況到底有多差」，而不只是「最差情況的邊界在哪」。

我們的 α = 0.85，關注最差 15% 的情況（比 95% 寬鬆，但覆蓋更多不利場景）。

### CPPO 的目標

```
PPO：  maximize E[回報]

CPPO： maximize E[回報]
       subject to CVaR_α ≥ 門檻

白話：盡量賺錢，但最差 15% 的情況不能太慘
```

這是一個**約束優化問題**（constrained optimization）。

### CPPO 的實現方式

不是用 solver 解約束問題，而是用 **Lagrangian 放鬆** 把約束融入 Advantage：

**Step 1：追蹤門檻 ν**

```
ν 是一個動態門檻，代表「低於這個回報就算差」
初始 ν = 0.1（很低的門檻）
每個 epoch 用最近 episode 的回報更新 ν
```

**Step 2：在 Advantage 上加懲罰**

```
每個 episode 結束時，計算風險調整後的回報 D：
  D = R_f × (episode_return + V(s) - r)

  R_f = portfolio 風險因子（LLM 風險分數加權）
  高風險倉位多 → R_f 大 → D 被放大 → 更容易觸發懲罰

如果 D < ν（這次回報低於門檻）：
  penalty = λ_cvar / (1 - α) × (ν - D)
  penalty = min(penalty, |V(s)| × cvar_clip_ratio)  ← 截斷防止過大

  A_t = A_t - penalty   ← Advantage 被壓低
```

效果：那些導致「差的 episode」的行動，Advantage 被額外壓低，
agent 會更強烈地避開這些行動。

**Step 3：自適應調整**

```
Lagrange 乘子 λ_cvar：控制懲罰強度
  每個 epoch：λ_cvar = λ_cvar + lr × (β - ν)
  β = 3000.0（目標約束值）

  如果 ν < β（門檻太低，風險太大）→ λ_cvar 增大 → 懲罰加重
  如果 ν > β（門檻足夠高）→ λ_cvar 減小 → 懲罰放鬆

門檻 ν：追蹤最近的回報水平
  ν = ν + lr × (D - ν)  ← 指數移動平均
```

### LLM 風險分數在 CPPO 裡的兩個作用

**作用 1：直接抑制高風險交易**

```python
# 在環境的 step() 裡
actions[risk_score >= 4] *= 0.8   # 高風險股票交易量打八折
```

這是硬性的，不管 agent 怎麼決策。

**作用 2：加重 CVaR 計算**

```
portfolio 風險因子 R_f = Σ(w_i × risk_weight_i)

w_i = 第 i 支股票的倉位比重
risk_weight_i 由風險分數決定：
  風險 1 → 0.99（幾乎不影響）
  風險 3 → 1.00（中性）
  風險 5 → 1.01（加重）

持有高風險股票多 → R_f 大 → episode 回報 D 被放大
→ 更容易低於門檻 ν → 觸發更多懲罰
→ agent 被迫減少高風險股票的倉位
```

## 第五步：PPO vs CPPO 的 Tradeoff

### 理論上的 tradeoff

```
        ↑ 期望回報
        │
        │    * PPO（無約束，回報高）
        │
        │         * CPPO（有約束，回報降低但風險可控）
        │
        │                * 保守策略（低回報低風險）
        │
        └────────────────────────────────→ 風險（CVaR 絕對值越小越好）
```

CPPO 犧牲一部分期望回報，換取尾部風險的控制。
**多犧牲多少取決於約束的嚴格程度**。

### 我們的實驗結果

```
EXP-001 PPO  (s42): Return +123.1%,  Sharpe 1.66,  CVaR -5.9%,  MDD -21.1%
EXP-002 CPPO (s0):  Return  +8.9%,   Sharpe 0.32,  CVaR -4.7%,  MDD -26.2%
```

CVaR 改善了（-4.7% vs -5.9%），但回報代價太大。
可能的原因：

1. **seed 不同**（最重要）：PPO=42, CPPO=0，不能直接比較
2. **β = 3000 太嚴格**：對 Polygon 數據來說約束可能過重
3. **風險分數覆蓋率只有 29%**：大部分日子沒有風險信號，風險權重都是中性的

### 可調的 CPPO 旋鈕

> 以下參數在 `training/cppo.py` 的 `cppo()` 函數裡。

| 參數 | 目前值 | 效果 | 調整方向 |
|------|--------|------|----------|
| **β** | 3000.0 | CVaR 約束目標。越大 = 約束越嚴 | 降低（如 1000）→ 放鬆約束，回報可能提高 |
| **α** | 0.85 | 關注最差多少 %。0.85 = 最差 15% | 提高到 0.95 → 只關注最差 5%（更聚焦極端） |
| **cvar_clip_ratio** | 0.05 | 懲罰截斷。防止單步懲罰過大 | 提高 → 允許更大懲罰 → 更嚴格 |
| **nu_lr** | 5e-4 | 門檻 ν 的學習率 | 降低 → ν 更新慢 → 約束更穩定 |
| **lam_lr** | 5e-4 | λ_cvar 的學習率 | 降低 → 懲罰強度變化慢 |
| risk_weights | ±1% | 風險分數的縮放幅度 | 加大 → 風險分數影響更強 |

### PPO 可調的旋鈕

| 參數 | 目前值 | 效果 | 調整方向 |
|------|--------|------|----------|
| **clip_ratio (ε)** | 0.7 | 每次更新最大幅度 | 降低（0.2-0.3）→ 更穩定但學得慢 |
| **pi_lr** | 3e-5 | Policy 學習率 | 提高 → 學更快但可能震盪 |
| **vf_lr** | 1e-4 | Value 學習率 | 通常不需要調（2-5x of pi_lr 是合理的） |
| **target_kl** | 0.35 | KL early stop 閾值 | 降低 → 更早停止 → 更穩定 |
| **steps_per_epoch** | 20000 | 每輪收集多少經驗 | 增大 → 更穩定但每輪更慢 |
| **gamma** | 0.995 | 折扣因子 | 降低 → 更重視近期回報 |
| **lam (λ)** | 0.95 | GAE 偏差-方差平衡 | 降低 → Advantage 更平滑但可能有偏 |
| **hid / l** | 512 / 2 | 網路大小 | 減小 → 防過擬合；增大 → 更強表達力 |
| sentiment_scale | strong (±10%) | 情緒信號強度 | weak (±0.1%) → 減弱情緒影響 |
